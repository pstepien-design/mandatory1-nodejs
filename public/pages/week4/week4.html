FILES PT 2

As I mentioned before, to work with the files it is important to specify the path by __dirname or express.static Thanks to that node.js knows where the file is located and we can utilise it. Let's say we have a folder public. Inside of the public folder we have file with pages. In pages we can find a folder of each page. The structure looks like this 

          |--public
          |
          |--pages
              |
              |--homepages
                    |
                    |--homepage.html
                    |--homepage.css
                    |--homepage.js

Thanks to the static path we are capable of accessing those files for example to specify stylesheet in out html file.
< link rel="stylesheet" href="/pages/homepage/homepage.css" />

SSR

SSR stands for Server Side Rendering. Instead of rendering the page on the client's side, SSR takes different approach. Page is render whenever the request is being send. Thanks ot that the performance is higher, user doesn't have to wait for JavaScript to load, thus the waiting time is shorter and page loads almost instantly.
To acheive that we can use filesync, to get the content of our html files instantly. const fs = require('fs');
const homepage = fs .readFileSync('./public/pages/homepage/homepage.html') .toString();
By specifying the file and converting it into string, we can quickly send in our request the page in a form of string, instead of the whole html file, which makes the rendering much faster.
app.get('/', (req, res) => { res.send(homePage); });
